# The DJ Who Loves Me

## 프로젝트 소개

차가운 세상에서 내 감정을 계속 바라봐 주는 존재가 있다면, 우리의 마음은 조금 더 따뜻해질 수 있다고 생각했습니다.

저희가 만든 서비스는 사용자가 언제 어디서나 자신의 감정에 따라 몸을 자유롭게 움직이면, 그 움직임에 맞춰 음악을 틀어주는 웹 기반 인터랙티브 서비스입니다.  
다르게 표현하면, “나를 사랑하는 DJ”입니다.

기존의 많은 댄스 기반 서비스는 음악이 먼저 나오고 사람이 그 박자와 분위기에 맞춰 움직입니다.  
반대로 저희는 사용자의 움직임을 먼저 읽고, 그 움직임에 맞는 음악이 공간을 채우도록 만들었습니다.

즉, 춤이 음악을 따라가는 것이 아니라, 음악이 춤을 따라오게 만드는 것이 이 프로젝트의 핵심입니다.

---

## 핵심 아이디어

- 사용자는 카메라 앞에서 자유롭게 춤을 춥니다.
- 시스템은 웹캠으로 사용자의 포즈를 인식합니다.
- 포즈 변화에서 움직임 특징값을 계산합니다.
- 계산된 특징을 바탕으로 현재 춤의 스타일을 분류합니다.
- 분류 결과에 맞는 음악을 즉시 재생합니다.

최종적으로는 집 안의 카메라가 사용자의 움직임을 읽고, 집 안 전체의 분위기와 바이브가 사용자의 감정에 따라 바뀌는 경험을 만드는 것을 목표로 했습니다.

---

## 주요 기능

### 1. 실시간 춤 인식
- 웹캠 입력을 받아 사용자의 전신 포즈를 추적합니다.
- MediaPipe Pose를 사용해 웹 브라우저에서 바로 포즈 랜드마크를 분석합니다.

### 2. 움직임 특징 추출
- 손목, 어깨, 상체 중심 좌표를 바탕으로 움직임 특징을 계산합니다.
- 대표적으로 `Energy`, `Bounce`, `Openness` 같은 값을 사용해 현재 움직임의 성격을 판단합니다.

### 3. 장르 분류
- 실시간으로 들어오는 움직임을 바탕으로 현재 춤을 `Pop`, `EDM`, `Waltz` 중 하나로 분류합니다.
- 발표용 데모에서는 각 장르가 명확하게 구분되도록 규칙 기반 분류를 적용했습니다.

### 4. 음악 자동 전환
- 분류된 장르에 맞는 YouTube 곡을 자동으로 재생합니다.
- 일부 트랙은 곡의 특징이 잘 드러나는 구간부터 시작하도록 설정했습니다.


### 5. 제스처 기반 정지
- 사용자가 두 팔로 X 포즈를 만들고 약 2초 유지하면 음악이 즉시 정지됩니다.
- 발표 시 직접 상호작용을 보여주기 좋은 기능입니다.

## 구현 방식

이번 프로젝트의 가장 큰 한계는 충분한 학습 데이터가 없었다는 점이었습니다.

그래서 새로운 AI 모델을 직접 학습시키기보다는, 포즈 랜드마크에서 움직임 특징을 직접 추출하고 이를 규칙 기반으로 분류하는 방식을 사용했습니다.  
즉, 범용적인 대규모 춤 분류 모델을 만든 것이 아니라, 제한된 시간 안에서 데모에 적합한 정확도를 확보하는 방향으로 설계했습니다.

저희는 실시간 서비스 구현과 발표 상황에서의 안정적인 동작을 우선 목표로 두었고, 그에 맞는 휴리스틱 기반 분류 구조를 선택했습니다.

---

## 협업 방식

저희는 3명이 GitHub 브랜치를 나누어 협업했습니다.

- 각자 다른 구현 방향을 맡아 개발
- 각 브랜치에서 직접 실행해 보며 결과 비교
- 가장 안정적으로 동작하는 구현을 기준으로 선택
- 최종적으로 하나의 결과물로 병합

즉, 한 사람이 전체를 만드는 방식이 아니라, 각자의 구현안을 실제로 검증하고 더 나은 결과를 채택하는 방식으로 진행했습니다.
